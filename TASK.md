# **Backend Developer Interview Case Study: RAG Chat Storage Microservice Background**

You are tasked with building a backend microservice to store chat histories generated by a

RAG (Retrieval-Augmented Generation) based chatbot system.

Each conversation between a user and an AI assistant should be securely saved along with any relevant retrieved context.

# **Objective**

Build a **production-ready backend microservice** that:

* Stores and manages **chat sessions** and **chat messages**.

* Supports **session management** features like **rename**, **mark as favorite**, and

  # **delete**.

* Provides APIs to **create sessions**, **add messages**, **retrieve messages**, and

  # **manage sessions**.

* Follows **best practices** for scalability, security, error handling, and configuration management.

# **Requirements**

**Core Functionalities**

* Start and maintain **chat sessions** for users.

* Save **messages** within a session, recording the sender, content, and optional context.
* Enable **renaming** of chat sessions.

* Allow sessions to be **marked or unmarked as favorite**.

* Allow **deletion** of a chat session and its associated messages.

* Provide a way to **retrieve the message history** of a chat session.

# **Technical Expectations**

* You are free to choose **any backend language, framework, and database**.

* Environment-specific configurations should be properly managed (e.g., through

  .env files).

* All APIs should be protected using **API key authentication** (read from environment variables).
* Implement **rate limiting** to prevent abuse of the APIs.

* # Add **centralized logging** and **global error handling**.

* The project must be **Dockerized** for local setup.

* Provide a **README** with setup instructions.

# **Deliverables**

* Source code in a GitHub repository.

* .env.example file documenting all environment variables used.

* README.md including:

  * Setup and running instructions.

  * A description of the available APIs.

  * Optional: Include API documentation (e.g., Swagger) if implemented.

# **Bonus**

* Add **health check** endpoints.

* Provide **Swagger/OpenAPI** documentation.

* Dockerize a database management tool (e.g., pgAdmin, Adminer) for easier browsing.
* Write basic **unit tests** for your services or business logic.

* Implement **CORS** configuration for security.

* Add pagination support when retrieving chat messages.
